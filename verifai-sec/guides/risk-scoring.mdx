---
title: Risk scoring
description: How VerifAI SEC's AI risk scoring system works
---

Every filing processed by VerifAI SEC receives a **risk score** from 0 to 100. The score reflects the number and severity of anomalies detected by our AI models.

## Score ranges

| Range   | Label      | Meaning                                                   |
| ------- | ---------- | --------------------------------------------------------- |
| 0–20    | Low        | Clean filing with no significant anomalies                |
| 21–40   | Moderate   | Minor irregularities worth reviewing                      |
| 41–60   | Elevated   | Notable anomalies that warrant investigation              |
| 61–80   | High       | Serious flags across multiple dimensions                  |
| 81–100  | Critical   | Major red flags — likely restatement, fraud, or material misrepresentation |

## What triggers risk flags

The AI evaluates filings across several dimensions:

### Financial anomalies

- Revenue recognition changes
- Unusual related-party transactions
- Material restatements or corrections
- Significant changes in accounting estimates
- Going concern qualifications

### Disclosure gaps

- Missing or incomplete risk factor disclosures
- Unusual changes to critical accounting policies
- Late or amended filings
- Significant redactions or omissions

### Pattern analysis

- Deviations from the company's historical filing patterns
- Industry peer comparison outliers
- Timing anomalies (e.g., filing delays near earnings)
- Changes in auditor or legal counsel

### Textual signals

- Sentiment shifts in MD&A (Management Discussion & Analysis)
- Increased use of hedging language
- Changes in forward-looking statement tone
- Unusual legal disclaimer additions

## Using risk scores in your application

### Filter filings by risk level

```bash
# Search for flagged filings
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://api.irora.ai/verifai-sec/v1/filings/search?q=tesla"
```

Check the `status` and `riskScore` fields in the response:

```json
{
  "company": "Tesla Inc.",
  "ticker": "TSLA",
  "status": "flagged",
  "riskScore": 67
}
```

### Status values

| Status     | Description                          |
| ---------- | ------------------------------------ |
| `verified` | Filing passed all checks (score 0-20) |
| `flagged`  | One or more anomalies detected (score 21+) |
| `pending`  | Filing is still being processed       |

<Note>
  Risk scores are AI-generated signals, not investment advice. Always perform
  your own due diligence before making investment decisions based on filing
  analysis.
</Note>
